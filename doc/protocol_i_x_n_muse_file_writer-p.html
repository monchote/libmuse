Detailed Description

Class manages saving Muse headband packets and other data into a file, which can later be read and replayed by MusePlayer. Google Protobuf is used to store information. For better control data is not written to the file immediately, but store them into a buffer instead. Buffer then can be flushed to a file or discarded.
Threading: It is thread safe, so you can call write/get/flush/discard operations from different threads.
Important: If you don't call flush (IXNMuseFileWriter-p) method, messages will be accumulated in memory and eventually you will get memory overflow error. It's library client responsibility to call flush (IXNMuseFileWriter-p) or discard() and to clean the memory. You can use helper methods to get how many messages are currently stored in the buffer and how much memory is used.

Method Documentation

- (void) addAnnotation: (int32_t)  id
annotation: (IXNAnnotationData *)  annotation 

Adds annotation data structure to the buffer. Similar to addAnnotationString, but this method allows you to extra fields. Empty fields in annotation struct won't be added to protobuf. If annotation.data field is empty, method returns immediately. id allows data from multiple devices to be recorded to a single file.

- (void) addAnnotationString: (int32_t)  id
annotation: (NSString *)  annotation 

Adds annotation string to the buffer. It may be useful if you want to log specific events. This is a simplified version of addAnnotation. Use this method if you want to add one string to protobuf. Use addAnnotation for more advanced options. id allows data from multiple devices to be recorded to a single file. Method does nothing if annotation string is empty.

- (void) addDataPacket: (int32_t)  id
packet: (IXNMuseDataPacket *)  packet 

Adds Muse Data Packet to the buffer. Not all Data Packets are supported. If unsupported data packet is passed, it will be ignored. All algorithm packets (ALPHA, BETA, HORSESHOE, etc) are ignored by this method. Support for these methods using new message in protobuf shcema will be added in next release. id allows data from multiple devices to be recorded to a single file.

- (void) open

Opens a file if it exists or creates a new one. If file exists, then puts pointer to the end of file, so all new messages will be appended at the end. Use this method if you explicitly closed file and want to open it again. Calls Muse File open.
